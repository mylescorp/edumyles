# EduMyles School Pilot Pack

This document is designed to be shared with schools.

## 1. Pilot Overview

EduMyles is inviting selected schools to join a pilot program for our school management platform built for East African schools.

The pilot is designed to help schools digitize key daily workflows in one place, including:

- student records
- class management
- staff access
- attendance
- exams and report preparation
- fee invoices and payment tracking
- parent access
- school announcements and communication

## 2. What Schools Will Get During the Pilot

- guided onboarding and setup support
- school-branded workspace
- admin access for school leadership
- teacher access for selected staff
- parent onboarding for selected families
- priority support during the pilot period
- weekly check-ins where needed

## 3. What We Need From the School

To start the pilot smoothly, each school should provide:

- one main school admin champion
- one backup contact person
- school logo and official school details
- class list
- student list
- parent or guardian contact list
- teacher and key staff list
- fee structure for the current term
- preferred go-live date

## 4. What the Pilot Will Cover

The pilot will focus on real day-to-day use, not just demos.

Suggested pilot scope:

- student enrollment and records
- class setup
- teacher onboarding
- attendance marking
- fee invoice generation
- payment tracking
- parent access
- school communication

## 5. Pilot Timeline

Recommended timeline:

- Week 1: kickoff, school data collection, setup
- Week 2: admin and teacher onboarding
- Week 3: parent onboarding and live usage
- Week 4 to 6: active pilot usage and support
- Final week: review, feedback, and conversion discussion

## 6. Pilot Success Metrics

The pilot will be considered successful when the school is actively using EduMyles for core workflows.

Typical success indicators:

- admin team logging in weekly
- students uploaded successfully
- staff invited and activated
- attendance marked in the system
- invoices created
- parents onboarded
- announcements sent
- school feedback collected

## 7. Support During Pilot

EduMyles will provide:

- onboarding support
- bug reporting and issue follow-up
- user guidance for school admins
- regular check-ins during the pilot window

The school should provide:

- a responsive pilot champion
- timely test feedback
- access to sample data where needed
- clear communication on issues and priorities

## 8. Pilot Conditions

- pilot access is limited to approved schools
- the pilot is for live evaluation of the platform
- some features may improve during the pilot period
- schools are expected to share practical feedback
- pilot scope and duration will be agreed before launch

## 9. School Decision Checklist

Before starting the pilot, the school should confirm:

- we have assigned a pilot champion
- we can provide required setup data
- we are ready to test with real users
- we are comfortable giving structured feedback
- we want to explore full rollout if the pilot succeeds

## 10. Contact

For onboarding, scheduling, or pilot questions, the EduMyles team will coordinate directly with the school administration team.
